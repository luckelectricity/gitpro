我在学习git

在想要使用git管理的文件上使用初始化git init

我想试试怎么用git add和git commit

git add 可以反复使用，添加多个文件

git commit必须加-m文字说明

git status让我们掌握仓库的状态

git diff 告诉我们你修改了什么

git log 可以查看从最近到最远的提交日志

git log --pretty=oneline简化提交日志

git reset --hard HEAD^ 返回上次提交，HEAD是本次版本，有几个^就是返回上几次，如果比较多，就用git reset --hard HEAD~100（比如返回100次）{回退的真狠}

git reflog 用来记录你的每一次操作，在git上你的操作也是被实时监控的。如果上一步有人回退的太多，根据commit id 你也可找到你想找到的那次提交

git add添加文件到暂存区，git commit才把文件添加到工作区，这就像购物车和付款一样，如果每买一次物品就付款，多麻烦啊。

如果输入错误，还没有git add 加文件名 ，就用git checkout -- 加文件名撤销提交，{我觉得在文件删了直接再提交就好了,如果直接删除了，记录里还有呢，并且多记录了删除的步骤}。

如果git add了，就用git reset HEAD 文件名退回到未add前，然后用git checkout -- 文件名删除就好了.

如果你提交了版本库，参考版本回退的那次吧，如果你提交了远程仓库，节哀顺变吧。

如果没有README.md文件，则需要把他下到本地文件才行，用git pull --rebase origin master来吧这个文件下载到本地

第一次提交远程仓库，用git push -u origin master >>>>>> 意思是提交的你仓库的master分支加上-u参数，不但会把本地的master分支提交到远程的master分支，还会把两个分支关联起来

去github上克隆仓库，git clone xxxxxxxxxxxxxxxxxxxxxx地址会把仓库克隆下来，cd XXX仓库名，进入仓库>>>>>>>>>> ls 查看仓库文件列表

分支控制

git checkout -b dev创建并转到dev分支，这是两条命令的集合，创建dev是git branch dev,切换到dev是git checkout dev

git branch查看当前分支，git checkout master 切换至主分支，git merge dev 讲dev分支的合并到主分支，git branch -d dev 删除dev分支
